# コアルール（常時適用）

## 1. 言語設定
- ユーザーとのやり取りは日本語で行う
- コードコメント・ドキュメントも日本語で記述
- コード自体（変数名、関数名、クラス名）は英語で記述

## 2. コード品質
- `declare(strict_types=1);` を全PHPファイル先頭に記述
- `readonly` を活用（コンストラクタ注入は `private readonly`）
- 早期リターンでネストを浅く保つ
- 1メソッド1責任
- ハードコーディング禁止（文字列・数値は定数化または翻訳キー化）

## 3. 実行環境
- Artisan/DB操作は Docker内で実行
- テスト実行も Docker内で実行
- ホスト上での `php artisan` 直接実行は禁止
- `php artisan tinker --execute=...` による検証は禁止
- bash heredoc によるファイル生成は禁止

## 4. テスト
- テスト作成後は必ず実行確認
- テストファイルは Artisan で雛形生成し、内容編集は通常のファイル編集で行う

## 5. Agent作業範囲
- ✅ ファイル作成・編集、Artisan Command作成・実行、テスト実行
- ⚠️ 既存マイグレーション、既存テストスイート
- ❌ 本番マイグレーション、`.env` 変更、`composer` 操作
